plan standar api

standar status respon:

200 - OK - api success
201 - CREATED - post success
400 - BED REQUEST - error on client side (bisa input yang salah dll)
401 - UNAUTORIZED - user not authorized to the REQUEST
403 - FORBIDDEN - user not allowed to acess
404 - NOT FOUND - request endpoin not FOUND
500 - INTERNAL SERVER ERROR - error on server side
502 - BAD GETEWAY - invalid respons from another request


{root.api}/{version}/{group}/{endpoin}
exemple:
http://localhost:3000/v1/auth/login

group : Authentication 

[1] - register - {root.api}/v1/auth/register

req: {
    method: post,
    data: {
        username: "admin"
        email: "test@gmail.com"
        password: "password"
    }
}

res: {
    "message" : "register success"
    data: {
        username,
        email,
        password
    }
} 

error.response:
400 - input yang anda masukan salah

[2] - Login - {root.api}/v1/auth/login

req: {
    method: post,
    data: {
        email: "test@gmail.com"
        password: "password"
    }
}

res: {
    "message" : "login success"
    data: {
        username,
        email,
        password
    }
} 

error.response:
400 - input yang anda masukan salah
401 - anda belum login
